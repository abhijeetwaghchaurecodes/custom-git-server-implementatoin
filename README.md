# SEBI Mutual Fund — Yaegi Injection Pipeline

Full end-to-end system demonstrating runtime implementation injection for a SEBI-compliant MF backend.

---

## Architecture

```
mf-sebi/
│
├── go.work                                     ← Go workspace (links all 3 modules)
│
├── templates/
│   └── bootstrap.go.gonja                      ← Jinja2-style template (Gonja)
│
├── project1-mf-backend/                        ← Echo HTTP server (STUBS ONLY)
│   ├── cmd/server/main.go                      ← Echo startup, calls bootstrap.RegisterAll()
│   ├── internal/
│   │   ├── models/models.go                    ← Full SEBI MF request/response types
│   │   ├── contracts/interfaces.go             ← 4 service interfaces (no impl)
│   │   ├── registry/registry.go                ← Register*/Get* injection points
│   │   ├── service/mf_service.go               ← Stubs: delegate to registry only
│   │   ├── handler/mf_handler.go               ← Echo handlers for all routes
│   │   └── bootstrap/bootstrap.go              ← AUTO-GENERATED by injector
│
├── project2-mf-implementations/                ← Real business logic (separate module)
│   ├── mfcreate/mf_create.go                   ← KYC, PAN, unit allotment, folio
│   ├── mftransfer/mf_transfer.go               ← Exit load, STCG/LTCG, switch
│   ├── mfupdate/mf_update.go                   ← OTP 2FA, SIP/bank/nominee update
│   └── mfdelete/mf_delete.go                   ← Lock-in, TDS, redemption payout
│
└── service-injector/                           ← The injection orchestrator
    ├── db/store.go                             ← SQLite: store + load records
    ├── seeder/seeder.go                        ← Read project2 source → store in DB
    ├── validator/yaegi_validator.go            ← Yaegi smoke-test
    ├── templategen/template_gen.go             ← Gonja template rendering
    ├── runner/runner.go                        ← go fmt, go build
    └── cmd/injector/main.go                   ← Main orchestrator (run this)
```

## API Routes

| Method   | Route                    | Handler       | Service stub  | Implementation    |
|----------|--------------------------|---------------|---------------|-------------------|
| `POST`   | `/mutualfund`            | `Create()`    | `MFCreate()`  | `MFCreateImpl`    |
| `POST`   | `/mutualfund/transfer`   | `Transfer()`  | `MFTransfer()`| `MFTransferImpl`  |
| `PUT`    | `/mutualfund`            | `Update()`    | `MFUpdate()`  | `MFUpdateImpl`    |
| `DELETE` | `/mutualfund`            | `Delete()`    | `MFDelete()`  | `MFDeleteImpl`    |

## Setup & Run

```bash
# 1. From mf-sebi/ root — initialize Go workspace
go work init
go work use ./project1-mf-backend
go work use ./project2-mf-implementations
go work use ./service-injector

# 2. Install project1 deps
cd project1-mf-backend
go get github.com/labstack/echo/v4
go mod tidy
cd ..

# 3. Install injector deps
cd service-injector
go get github.com/nikolalohinski/gonja/v2
go get github.com/traefik/yaegi@latest
go get modernc.org/sqlite
go mod tidy

# 4. Run the injector (does steps 3-6 automatically)
go run ./cmd/injector

# 5. Start the server
cd ../project1-mf-backend
go run ./cmd/server
```

## What the Injector Does

```
project2 source files
       │
       ▼
  [seeder]  reads .go source + sha256
       │
       ▼
  [SQLite]  stores source + metadata (status=validated)
       │
       ▼
  [Yaegi]   smoke-test: parse + symbol check
       │
       ▼
  [Gonja]   renders bootstrap.go.gonja with DB records as Jinja2 context
       │
       ▼
  project1/internal/bootstrap/bootstrap.go  ← written/overwritten
       │
       ▼
  go build ./...  ← compiler validates interface compliance
       │
       ▼
  go run ./cmd/server  ← all 4 routes live
```

## SEBI Compliance Notes

| Function      | SEBI Rules Implemented                                                  |
|---------------|-------------------------------------------------------------------------|
| `MFCreate`    | KYC=VERIFIED mandatory, PAN format (AAAAA9999A), AMFI min amounts, T+0/T+1/T+2 allotment |
| `MFTransfer`  | Same-AMC restriction for switch, exit load, STCG/LTCG flags, T+2/T+3 settlement |
| `MFUpdate`    | OTP 2FA mandatory, SEBI-allowed SIP dates (1/5/10/15/20/25), IFSC validation |
| `MFDelete`    | OTP 2FA, lock-in check, TDS deduction, payout to registered bank only, T+3 settlement |
